# Design System Document

This document outlines the core aesthetic and functional principles of our design system, based on the current theme settings.

## 1. Theme Configuration

Our design system is currently configured with the following global theme settings:

*   **Color Mode:** Light
*   **Roundedness:** Moderate (level 2)
*   **Spacing:** Normal (level 2)

## 2. Color Palette

Our brand colors are carefully selected to create a cohesive and engaging user experience.

*   **Primary Color:** `#0082fd` (A bright, energetic blue, ideal for primary calls to action and key interactive elements.)
*   **Secondary Color:** `#5071d5` (A softer, supportive blue, suitable for less prominent UI elements, chips, and secondary actions.)
*   **Tertiary Color:** `#a459b5` (A vibrant purple, used for highlights, badges, or decorative elements.)
*   **Neutral Color:** `#6a769e` (A muted bluish-gray, forming the base for backgrounds, surfaces, and non-chromatic elements.)

## 3. Typography

The chosen typefaces ensure readability and reinforce our brand's voice across all applications.

*   **Headline Font:** Inter (A modern, highly legible sans-serif font, providing a clean and contemporary feel for headings.)
*   **Body Font:** Inter (The same versatile Inter font is used for body text, ensuring consistent readability and harmony with headlines.)
*   **Label Font:** Inter (Inter is also adopted for labels, maintaining a unified typographic voice across interactive elements.)

## 4. UI Elements (General Principles)

### Borders & Corners

User interface elements feature a **moderate roundedness (level 2)**, striking a balance between modern aesthetics and approachability without being overly soft or sharp.

### Spacing

A **normal spacing (level 2)** approach is implemented, providing a comfortable amount of whitespace between and within UI elements, contributing to a balanced and organized layout without feeling too dense or sparse.

## 5. Development Guidelines

All design token values derived from these theme settings are available for developers. Consistency with these values is crucial for maintaining the integrity and scalability of the design system. Refer to the generated design tokens for specific values.
